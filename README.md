# dependencies_edu
##1. Общее описание:
   Инструмент визуализации шрафа зависимостей для менеджера пакетов.
##2. Описание функций и настроек:
   Класс Config - хранит все поля конфига.
   Функция build_dependency_graph(config, packages_content) - рекурсивно строит граф зависимостей пакетов с поиском в глубину. Возвращает словарь, представляющий гираф зависимостей. Парамерты: config - обьект конфига, packages_content - содержимое packages.
   Функция parse_toml_value(value_str) - кастомный парсер .toml. Возвращает значение из файла в нужном формате. Параметры: value_str - строковое значение из файла конфига.
   Функция parse_depends_line(depends_line) - парсер сложных строк запвисимостей. Возвращает список чистых имен пакетов. Параметры: depends_line - строка из свойства зависимостей файла packages.
   Функция find_package_dependencies(packages_content, package_name, package_version) - ищет зависимости пакета версии в файле packages. Возвращает список зависимостей. Параметры: packages_content - содержимое файла packages, package_name - имя искомого пакета, package_version - искомая версия пакета
   Функция load_config(filename) - загружает конфиг из файла. Возвращает обьект Config с заполненными полями. Параметры: filename - путь к файлу конфига.
   Функция get_latest_package_version(packages_content, package_name) - определяет последнюю версию пакета. Возвращает строку с последней версией пакета. Параметры: packages_content - содержимое packages, package_name - имя пакета.
   Функция get_packages_content(repository_url, test_repo_mode) - Получает содержимое файла пакетов. Возвращает содержимое файла packages. Параметры: repository_url - URL или путь к локальному репозиторию, test_repo_mode - флаг режима работы(тестовый или нет).
   Функция print_dependency_graph(graph, config) - выводит граф зависимостей в читаемом формате. Параметры: graph - словарь с графом зависимостей, config - обьект конфигурации.
   Функция main() - основной алгоритм программы.
##3. Описание команд для сборки проекта и запуска тестов:
   Требования:
     - Python 3.6 или выше
     - Доступ к интернету (для работы с сетевыми репозиториями)
     - Права на чтение/запись файлов (для тестового режима)
     - Скачанная папка должна находиться в папке загрузок на диске C.
   Установка и настройка:
      1. Скачать main.py
      2. По необходимости заменить путь к конфигу по умолчанию(указан в строке 298)
   Запуск:
      1. Запустить PowerShell
      2. Перейти к директории, содержащей main.py с помощью команды cd <директория>
      3. Ввести команду "python main.py" При необходимости использования определенного файла конфига путь к нему в качестве параметра команды - "python main.py <путь к файлу>". ПУТЬ ВВОДИТЬ БЕЗ КАВЫЧЕК.
##4. Примеры использования:
   Стандартное использование: просто перейти к директории и ввести команду "python main.py".
   Тестирование: перейти к директории и ввести команду "python main.py" с аргументом "C:\Downloads\conf\configTest.toml". Программа запустится в тестовом режиме.
